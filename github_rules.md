# GitHubの運用ルール

* version 0.0.1​

## 1. 本ドキュメントの目的

* 本文書では，Organization`medical-rocker`におけるリポジトリ管理方法を定め，プロジェクトの円滑な運用を実現する．また，トラブル等の発生を防ぐ．
* GitHubの基本的な使用方法については，スライド[医療データ解析者へ向けた Git・GitHub 入門](https://www.slideshare.net/YuiTomo1/gitgithub-238968731)などを参照されたい．

## 2. 基本ルール

### 2.1. リポジトリの命名

* リポジトリの命名は，**半角英数字**を用いる．
* 特別な理由がない限り，**英字は小文字**を用いる．
* 語を区切る際には，**ハイフン(-)を用い，アンダースコア(_)は用いない**．

### 2.2. リポジトリの管理

#### 2.2.1. ブランチに関するルール

* `main(master)`ブランチから`develop`ブランチを切り，`develop`ブランチから機能追加や修正などを目的とした作業用ブランチを切る．
* ソフトウェアに関するリポジトリでは，`main(master)`ブランチでは，基本的に**テストが終了し問題なく実行できる状態**となるようにする．
  * `main(master)`ブランチには、`develop`ブランチからのみmergeを行う．
  * `main(master)`ブランチおよび`develop`ブランチには直接pushしない．
  * プロジェクト開始初期に，ディレクトリ構成やREADMEを作成した雛形を`main(master)`ブランチへpushしておいてもよい．
* ドキュメントに関するリポジトリでは，`main(master)`ブランチに置かれているファイルの内容が効力を有するとする．
* 各作業用ブランチからのmergeは，`develop`ブランチにて行う．
  * 各作業用ブランチからのpull requestは，`develop`ブランチへ向けて行う．

### 2.3. READMEの作成

* `README.md`というファイルをリポジトリ内に作成し，主なドキュメントは`README.md`に記すこととする．
* READMEの執筆は，日本語または英語で行う．
* READMEの構成は，必要に応じて下記を含める．
  * **リポジトリの説明（必須）**
  * **使用方法**
    * 依存環境を含む，起動方法に関する説明，およびコマンドなど．
  * **各ディレクトリ（あるいはファイル）についての説明**
  * **ライセンス**

### 2.4. .gitignoreの使用

* `.gitignore`とは，gitの管理対象外とするファイルを設定するファイルであり，不要なファイルを誤ってGitHubへアップロードしてしまうことを防ぐ等の観点から，作成を推奨する．
* 不要なファイル群や機密性の高いファイル群などを誤ってアップロードするリスクを低減するため，`.gitignore`の記述方法として，特別な理由がない限り**ホワイトリスト方式**を採用する．
  * ホワイトリスト方式では，まず`.gitignore`の最上部にワイルドカード(`*`)を記述し，全てを管理対象から外す．
  * 続いて，管理対象とするディレクトリやファイルを許可する記述を行う．
    * 例えば拡張子`.md`のファイル管理を許可するリポジトリとするためには，下記のように`.gitignore`ファイルへ記述すればよい．

  ```.gitignore
  # ignore all directories and files except for this repository
  *

  # allow files below
  !*.md
  ```
